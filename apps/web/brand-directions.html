Brand System Exploration

不要再用 CC 当 logo 了

这个产品要表达的不是两个字母，而是一个本地 workspace studio: 你配置一次，然后一键启动 terminal、agent、review、docs、doctor。 Logo 应该表达「启动」「分支」「控制台」「多窗口编排」。

1主品牌应该像本地控制台，不像云 SaaS，也不像黑客玩具。
2App logo 不能依赖文字。收起 sidebar 到 38px 时仍然要识别。
3项目 logo 可以继续用 initials 和颜色，因为那是项目身份，不是 app 品牌。

Logo 方向

我会避免用 CC 字母。下面这些图形都能在 sidebar 收起后当 app mark 使用，项目本身再用彩色 initials。

Launcher Loop

推荐。外圈是 console/studio，竖线是 power/launch，底部两点代表 workspace 分支。表达「一键启动多个工作区」。

Fit 9.3

Console Window

最直接的命令行表达。好处是清楚，风险是太像普通 terminal app，缺少「编排」这层含义。

Fit 7.7

Branch Runner

强调 branch / session 关系。适合 cc-branch 这个名字，但它对「一键启动」的表达不如 Launcher Loop。

Fit 8.2

Slot Stack

表达 slots / windows 编排。作为产品图标很准确，但更像功能 icon，不够像品牌标志。

Fit 8.0

Play Branch

最强的「启动」暗示。问题是 play 三角容易让人想到 media app，需要结合 terminal UI 才不会跑偏。

Fit 7.9

Workspace Orbit

表达多个 agent / terminal 围绕一个项目运转。漂亮，但偏抽象，容易变成普通 AI 产品视觉。

Fit 7.2

第二轮，更像品牌

上一轮的问题是太像功能图标。第二轮把抽象来源收窄到三个东西: sidebar dock、workspace lanes、一键启动。这样更贴产品，也更容易做成长期品牌。

Launch Rail

我现在最推荐这个。左边竖条就是收起后的 project sidebar，右边三角是启动。 它把你的核心交互变成 logo: 选择项目，然后一键启动 workspace。

Fit 9.5

Workspace Lanes

最像这个产品的真实结构: 一个项目入口，展开出多条 terminal / agent lane。 小尺寸也清楚，适合 app icon、favicon、sidebar 顶部。

Fit 9.2

Command Gate

像一个进入 workspace 的门，也像命令行里的执行箭头。更有品牌感， 但和 sidebar / project 模型的联系不如 Launch Rail 直接。

Fit 8.4

Console Deck

表达多个 terminal pane 叠在一个本地 studio 里。很适合桌面端， 但作为小 favicon 时层次可能损失。

Fit 8.1

Prompt Spark

表达从一个命令触发多个会话。更有动势，也更像 startup 工具。 风险是略抽象，需要配合产品 UI 才能看懂。

Fit 8.0

Switchboard

最像控制台和编排器。意思准确，但可能太像 settings / mixer。 适合做插画或空状态，不一定适合做主 logo。

Fit 7.6

主题色选择

主题色不是挑好看颜色。它要回答用户打开 app 时的感受：这是一个可靠的本地控制台，还是一个花哨网页。

Graphite + Signal Teal

推荐。最像 local-first console，也有足够品牌识别。

Workspace Lanes 深化

这个方向可以更好看。关键是不要让三条 lane 全都同色。 品牌骨架保持 graphite + teal，局部用 amber、blue、green 表达不同 workspace 状态。

Signal Spine

我最推荐这个版本。左侧 spine 是项目 dock，三条 lane 有不同状态色。结构稳，颜色不死板，小尺寸也清楚。
