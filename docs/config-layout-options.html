Config Editor Layout Lab

把 Workspace 放到第一层，Project / Display / Agents 退到第二层。

这页不是最终产品页，而是给你看配置页的几种信息架构方向。 核心问题不是“能不能放下”，而是“用户第一眼先看到什么、先理解什么、再去改什么”。

1 第一眼先看到 Workspace，而不是一堆同级卡片。
2 常用配置和高级配置分开，避免一次展开太多字段。
3 Project / Display / Agents 变成“工作区内的模块”，不是抢主位的主角。
4 每个方案都保留快速预览、同步写入和可编辑态。

我建议你优先看这三个方向

  • Workspace 头部总览 + 下方分区卡片。最稳，最适合现在的产品。
  • 左侧 Workspace 导航 + 右侧内容面板。更像控制台，信息密度高。
  • 分步骤编辑，但保留实时预览。适合新用户第一次配置。
当前的目标不是“把 UI 做得更花”，而是把层级改清楚。先让用户知道自己在配置什么，再让他改字段。
A. Workspace First B. Section Bands C. Left Rail D. Wizard + Preview

方案 A: Workspace First

把 Workspace 当成一级信息，Project / Display / Agents / Slots 变成下面的模块卡。打开页面先看总览，再往下展开具体配置。这个版本最接近你现在的产品，但层级更清楚。

方案 C: Left Rail Workspace

左侧做一个 Workspace 导航和摘要，右侧是编辑区。对比当前“所有东西一锅端”的方式，这个方案更像一个真正的控制台，适合复杂项目。

方案 D: Wizard + Preview

给第一次接触的人用。先选择 Workspace，再进入 Project / Display / Agents / Slots 的步骤流。右侧始终显示同步预览，这样用户不会陷入“我改了什么”的盲区。
